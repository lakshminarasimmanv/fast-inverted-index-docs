Configuration Options

Fast-Inverted-Index provides multiple ways to configure the index to suit different use cases and performance requirements. This guide covers the available configuration options and demonstrates how to use them effectively.

Overview​

There are three main approaches to configuring the index:

  1. Basic Constructor: Simple and convenient for common scenarios
  2. Builder Pattern: Fluent API for complex configuration
  3. Configuration Objects: Fine-grained control over all aspects

Configuration Parameters​

Configuration in Python​

Basic Constructor​

The simplest way to configure the index is through the constructor:

from fast_inverted_index import Index

# Create with default settings
index = Index()

# Create with custom settings
index = Index(
    storage_path="/path/to/index",
    cache_size=100000,
    cache_ttl_secs=3600,
    store_positions=True,
    in_memory=False,
)

Builder Pattern​

For more complex configuration, use the builder pattern:

from fast_inverted_index import Index, DictionaryConfig

# Create a builder
builder = Index.builder()

# Configure the index
builder.with_storage_path("/path/to/index")
builder.with_cache_size(100000) 
builder.with_cache_ttl_secs(3600)
builder.with_store_positions(True)
builder.with_in_memory(False)
builder.with_max_suggestions(20)
builder.with_suggestion_max_distance(2)
builder.with_compact_on_close(True)

# Configure dictionary if needed
dict_config = DictionaryConfig(
    hot_terms_capacity=5000,
    hot_promotion_threshold=50,
    medium_promotion_threshold=3,
    optimization_interval_secs=1800
)
builder.with_dictionary_config(dict_config)

# Build the index
index = builder.build()

Configuration Objects​

You can also use the configuration objects directly:

from fast_inverted_index import Index, IndexConfig, DictionaryConfig

# Create dictionary configuration
dict_config = DictionaryConfig(
    hot_terms_capacity=5000,
    hot_promotion_threshold=50,
    medium_promotion_threshold=3,
    optimization_interval_secs=1800
)

# Create index configuration
config = IndexConfig(
    storage_path="/path/to/index",
    cache_size=100000,
    cache_ttl_secs=3600,
    store_positions=True,
    in_memory=False,
    max_suggestions=20,
    suggestion_max_distance=2,
    compact_on_close=True,
    dictionary_config=dict_config
)

# Create index with config
index = Index.with_config(config)

Configuration in Rust​

Basic Construction with IndexBuilder​

use fast_inverted_index::{IndexBuilder, Index};
use std::time::Duration;

// Create with builder (recommended approach)
let index = IndexBuilder::new()
    .with_storage_path("/path/to/index")
    .with_cache_size(100_000)
    .with_cache_ttl(Duration::from_secs(3600))
    .with_store_positions(true)
    .with_in_memory(false)
    .with_max_suggestions(20)
    .with_suggestion_max_distance(2)
    .with_compact_on_close(true)
    .build()?;

With Dictionary Configuration​

use fast_inverted_index::{IndexBuilder, Index, dictionary::HybridDictionaryConfig};
use std::time::Duration;

// Create dictionary configuration
let dict_config = HybridDictionaryConfig {
    hot_terms_capacity: 5000,
    hot_promotion_threshold: 50,
    hot_demotion_threshold: 10,
    medium_promotion_threshold: 3,
    medium_demotion_threshold: 1,
    optimization_interval_secs: 1800,
};

// Use in builder
let index = IndexBuilder::new()
    .with_storage_path("/path/to/index")
    .with_cache_size(100_000)
    .with_dictionary_config(dict_config)
    .build()?;

Best Practices​

  1. Memory-Sensitive Environments:

    • Reduce cache_size
    • Reduce hot_terms_capacity
    • Use persistent storage (set in_memory=false)

  2. Performance-Critical Applications:

    • Increase cache_size
    • Increase hot_terms_capacity
    • Consider in-memory mode for small-to-medium datasets
    • Adjust dictionary thresholds for your access patterns

  3. Balanced Approach:

    • Use default settings as a starting point
    • Benchmark with your specific workload
    • Incrementally adjust parameters based on observed performance

  4. Suggestion Quality:

    • Increase max_suggestions for more comprehensive autocomplete
    • Adjust suggestion_max_distance for fuzzy matching tolerance

Example Configurations​

Small In-Memory Search Engine​

# For small datasets (e.g., < 100K documents)
index = Index(in_memory=True, cache_size=5000)

Large Document Repository​

# For large datasets with optimal memory usage
builder = Index.builder()
builder.with_storage_path("/path/to/large_index")
builder.with_cache_size(500000)
builder.with_cache_ttl_secs(1800)  # Shorter TTL for freshness
builder.with_store_positions(False)  # Save space if phrase queries aren't needed
index = builder.build()

Autocomplete-Optimized​

# For search with heavy autocomplete usage
dict_config = DictionaryConfig(
    hot_terms_capacity=20000,  # More terms in hot layer
    hot_promotion_threshold=30,  # Easier promotion to hot layer
    medium_promotion_threshold=2  # Easier promotion to medium layer
)

builder = Index.builder()
builder.with_max_suggestions(30)  # More suggestions
builder.with_suggestion_max_distance(3)  # More fuzzy matching tolerance
builder.with_dictionary_config(dict_config)
index = builder.build()

Multi-Core System Optimization​

# For systems with many CPU cores (8+)
index = Index(
    cache_size=1000000,  # Very large cache
    in_memory=True
)

# Index documents in parallel
documents = [(i, f"Document {i} content") for i in range(10000)]
index.add_documents_parallel(
    documents, 
    num_threads=None,  # Auto-detect core count
    batch_size=500
)

By carefully configuring your index based on your specific requirements, you can achieve significant performance improvements and better search functionality.
