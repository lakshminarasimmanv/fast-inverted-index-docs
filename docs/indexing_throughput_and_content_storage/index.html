Maximizing Indexing Throughput with Content Storage Integration

This guide provides comprehensive strategies and practical examples for achieving maximum indexing throughput with Fast Inverted Index while efficiently integrating with external content storage systems. It covers optimizations for index creation, content storage integration, and system-level performance tuning.

Overview​

Fast Inverted Index is designed to store the inverted index (terms to document mappings) separate from document content. This architectural separation offers flexibility and specialization but requires careful integration for optimal performance. This guide will help you achieve high-performance indexing while maintaining this separation of concerns.

RocksDB Optimization for Maximum Indexing Speed​

The storage layer configuration is critical for high-throughput indexing. Our optimized RocksDB implementation provides several features specifically designed for bulk loading.

Enabling Bulk Loading Mode​

For existing indexes, you can enable bulk loading mode before large operations:

// Rust - Enable bulk loading mode for an existing store

    // Enable bulk loading mode (significantly improves write performance)

    // Disable bulk loading mode when finished (for durability)
    storage.set_bulk_loading_mode(false);
}

In Python, this would be done via the environment variables before creating the index.

Content Store Integration Patterns​

Direct Integration​

// Rust - Direct content store integration
struct IntegratedSearch {
    index: Index,
    content_store: ContentStore,
}

impl IntegratedSearch {
    // Add a document with content
    pub fn add_document(&mut self, doc_id: DocId, content: &str, metadata: &Metadata) -> Result<()> {
        // 1. Store content first
        self.content_store.store_document(doc_id, content, metadata)?;
        
        // 2. Index the document (content is tokenized but not stored)
        self.index.add_document(doc_id, content, metadata)?;
        
        Ok(())
    }
    
    // Search and retrieve content
    pub fn search(&self, query: &str) -> Result<Vec<(DocId, f32, String)>> {
        // 1. Search the index
        let results = self.index.search(query)?;
        
        // 2. Retrieve content for matches
        let mut full_results = Vec::with_capacity(results.len());
        for (doc_id, score) in results {
            if let Some(content) = self.content_store.get_document(doc_id)? {
                full_results.push((doc_id, score, content));
            }
        }
        
        Ok(full_results)
    }
}

        return full_results

Pipeline Integration​

For more complex scenarios, use a processing pipeline:

// Rust - Pipeline pattern with queuing
use std::sync::mpsc::{channel, Sender, Receiver};
use std::thread;

struct IndexingPipeline {
    content_store: ContentStore,
    index: Index,
    sender: Sender<(DocId, String, Metadata)>,
}

impl IndexingPipeline {
    pub fn new(content_store: ContentStore, index: Index) -> Self {
        let (sender, receiver) = channel();
        
        // Start indexing worker thread
        let index_worker = index.clone();
        thread::spawn(move || {
            // Process documents from the queue
            for (doc_id, content, metadata) in receiver {
                if let Err(e) = index_worker.add_document(doc_id, &content, &metadata) {
                    eprintln!("Error indexing document {}: {}", doc_id, e);
                }
            }
        });
        
        Self { content_store, index, sender }
    }
    
    pub fn add_document(&mut self, doc_id: DocId, content: &str, metadata: &Metadata) -> Result<()> {
        // 1. Store content immediately
        self.content_store.store_document(doc_id, content, metadata)?;
        
        // 2. Queue for asynchronous indexing
        self.sender.send((doc_id, content.to_string(), metadata.clone()))?;
        
        Ok(())
    }
}
# Python - Pipeline pattern with queuing
import threading
import queue

class IndexingPipeline:
    def __init__(self, content_store, index, queue_size=10000):
        self.content_store = content_store
        self.index = index
        self.queue = queue.Queue(maxsize=queue_size)
        
        # Start worker thread
        self.worker = threading.Thread(target=self._indexing_worker, daemon=True)
        self.worker.start()
        
    def _indexing_worker(self):
        while True:
            try:
                # Get document from queue
                doc_id, content, metadata = self.queue.get()
                
                # Index the document
                self.index.add_document(doc_id, content, metadata)
                
                # Mark task as done
                self.queue.task_done()
            except Exception as e:
                print(f"Error in indexing worker: {e}")
    
    def add_document(self, doc_id, content, metadata=None):
        # Store content immediately
        self.content_store.store_document(doc_id, content, metadata)
        
        # Queue for asynchronous indexing
        self.queue.put((doc_id, content, metadata))
        
    def wait_completion(self):
        # Wait for all queued documents to be indexed
        self.queue.join()

Multi-stage Batching​

For very large document collections, use multi-stage batching:

// Rust - Multi-stage batching
const BATCH_SIZE: usize = 10000;

pub fn index_large_collection<I, S>(
    index: &mut Index,
    documents: I
) -> Result<()>
where
    I: Iterator<Item = (DocId, String, HashMap<String, String>)>,
{
    let mut batch = Vec::with_capacity(BATCH_SIZE);
    let mut total_indexed = 0;
    
    // Enable bulk loading mode if using RocksDB
    if let Some(storage) = index.storage_as_rocksdb_mut() {
        storage.set_bulk_loading_mode(true);
    }
    
    // Process documents in batches
    for document in documents {
        batch.push(document);
        
        // When batch is full, index and create a new batch
        if batch.len() >= BATCH_SIZE {
            index.add_documents_batch(&batch)?;
            total_indexed += batch.len();
            println!("Indexed {} documents so far", total_indexed);
            batch.clear();
        }
    }
    
    // Index remaining documents
    if !batch.is_empty() {
        index.add_documents_batch(&batch)?;
        total_indexed += batch.len();
    }
    
    // Disable bulk loading mode and optimize
    if let Some(storage) = index.storage_as_rocksdb_mut() {
        storage.set_bulk_loading_mode(false);
    }
    
    println!("Completed indexing {} documents", total_indexed);
    Ok(())
}

Parallel Indexing Strategies​

Thread Pool for Content Retrieval​

// Rust - Thread pool for content retrieval
use rayon::prelude::*;

pub fn parallel_retrieve_and_index<S: ContentStore + Sync>(
    content_store: &S,
    index: &mut Index,
    doc_ids: &[DocId],
    num_threads: usize
) -> Result<()> {
    // Split doc_ids into chunks
    let chunk_size = (doc_ids.len() + num_threads - 1) / num_threads;
    let chunks: Vec<_> = doc_ids.chunks(chunk_size).collect();
    
    // Process chunks in parallel
    let results: Result<Vec<Vec<_>>> = chunks.par_iter()
        .map(|chunk| {
            // Retrieve documents for this chunk
            content_store.batch_get_documents(chunk)
        })
        .collect();
    
    // Flatten results
    let documents = results?.into_iter().flatten().collect::<Vec<_>>();
    
    // Index all documents in a single batch
    index.add_documents_batch(documents)?;
    
    Ok(())
}
# Python - Thread pool for content retrieval
import concurrent.futures

def parallel_retrieve_and_index(content_store, index, doc_ids, num_threads=8):
    # Split doc_ids into chunks
    chunk_size = (len(doc_ids) + num_threads - 1) // num_threads
    chunks = [doc_ids[i:i+chunk_size] for i in range(0, len(doc_ids), chunk_size)]
    
    # Function to process a chunk
    def process_chunk(chunk):
        return content_store.batch_get_documents(chunk)
    
    # Process chunks in parallel
    documents = []
    with concurrent.futures.ThreadPoolExecutor(max_workers=num_threads) as executor:
        future_to_chunk = {executor.submit(process_chunk, chunk): chunk for chunk in chunks}
        for future in concurrent.futures.as_completed(future_to_chunk):
            documents.extend(future.result())
    
    # Prepare for indexing
    batch = [(doc_id, {"content": content, **metadata}) for doc_id, content, metadata in documents]
    
    # Index in a single batch operation
    index.add_documents_with_fields_parallel(batch)

Producer-Consumer Pipeline​

// Rust - Producer-consumer pipeline
use std::sync::mpsc::{channel, Sender, Receiver};
use std::thread;
use std::sync::{Arc, Mutex};

pub struct IndexingPipeline {
    content_queue: Sender<DocId>,
    document_queue: Arc<Mutex<Vec<(DocId, String, HashMap<String, String>)>>>,
    max_batch_size: usize,
    content_store: Arc<dyn ContentStore + Send + Sync>,
    index: Arc<Mutex<Index>>,
}

impl IndexingPipeline {
    pub fn new(
        content_store: Arc<dyn ContentStore + Send + Sync>,
        index: Index,
        num_threads: usize,
        max_batch_size: usize
    ) -> Self {
        let (content_sender, content_receiver) = channel();
        let document_queue = Arc::new(Mutex::new(Vec::with_capacity(max_batch_size)));
        let index = Arc::new(Mutex::new(index));
        
        // Start content retrieval workers
        for _ in 0..num_threads {
            let content_store = Arc::clone(&content_store);
            let document_queue = Arc::clone(&document_queue);
            let content_receiver = content_receiver.clone();
            let index = Arc::clone(&index);
            let max_batch_size = max_batch_size;
            
            thread::spawn(move || {
                for doc_id in content_receiver {
                    // Retrieve document
                    if let Ok((content, metadata)) = content_store.get_document(doc_id) {
                        // Add to document queue
                        {
                            let mut queue = document_queue.lock().unwrap();
                            queue.push((doc_id, content, metadata));
                            
                            // When batch is full, index and clear
                            if queue.len() >= max_batch_size {
                                let batch = std::mem::replace(
                                    &mut *queue, 
                                    Vec::with_capacity(max_batch_size)
                                );
                                
                                // Release lock before indexing
                                drop(queue);
                                
                                // Index the batch
                                if let Ok(mut index_guard) = index.lock() {
                                    let _ = index_guard.add_documents_batch(&batch);
                                }
                            }
                        }
                    }
                }
            });
        }
        
        Self {
            content_queue: content_sender,
            document_queue,
            max_batch_size,
            content_store,
            index,
        }
    }
    
    pub fn queue_document(&self, doc_id: DocId) -> Result<()> {
        self.content_queue.send(doc_id)?;
        Ok(())
    }
    
    pub fn flush(&self) -> Result<()> {
        // Index any remaining documents
        let batch = {
            let mut queue = self.document_queue.lock().unwrap();
            std::mem::replace(&mut *queue, Vec::with_capacity(self.max_batch_size))
        };
        
        if !batch.is_empty() {
            let mut index = self.index.lock().unwrap();
            index.add_documents_batch(&batch)?;
        }
        
        Ok(())
    }
}
# Python - Producer-consumer pipeline
import threading
import queue

class IndexingPipeline:
    def __init__(self, content_store, index, num_threads=8, max_batch_size=10000):
        self.content_store = content_store
        self.index = index
        self.max_batch_size = max_batch_size
        
        # Queues
        self.id_queue = queue.Queue()
        self.doc_queue = queue.Queue()
        
        # Control flags
        self.shutdown = threading.Event()
        
        # Start content retrieval workers
        self.content_workers = []
        for _ in range(num_threads):
            worker = threading.Thread(
                target=self._content_worker,
                daemon=True
            )
            worker.start()
            self.content_workers.append(worker)
        
        # Start indexing worker
        self.indexing_worker = threading.Thread(
            target=self._indexing_worker,
            daemon=True
        )
        self.indexing_worker.start()
    
    def _content_worker(self):
        while not self.shutdown.is_set():
            try:
                # Get document ID from queue with timeout
                doc_id = self.id_queue.get(timeout=0.1)
                
                # Retrieve document
                content, metadata = self.content_store.get_document(doc_id)
                
                # Add to document queue
                self.doc_queue.put((doc_id, content, metadata))
                
                # Mark task as done
                self.id_queue.task_done()
            except queue.Empty:
                continue
            except Exception as e:
                print(f"Error in content worker: {e}")
    
    def _indexing_worker(self):
        batch = []
        
        while not self.shutdown.is_set():
            try:
                # Get document from queue with timeout
                doc_id, content, metadata = self.doc_queue.get(timeout=0.1)
                
                # Add to batch
                batch.append((doc_id, {"content": content, **metadata}))
                
                # Mark task as done
                self.doc_queue.task_done()
                
                # When batch is full or queue is empty, index and create new batch
                if len(batch) >= self.max_batch_size or self.doc_queue.empty():
                    if batch:
                        self.index.add_documents_with_fields_parallel(batch)
                        batch = []
            except queue.Empty:
                # Index any documents in the batch
                if batch:
                    self.index.add_documents_with_fields_parallel(batch)
                    batch = []
            except Exception as e:
                print(f"Error in indexing worker: {e}")
    
    def queue_document(self, doc_id):
        self.id_queue.put(doc_id)
    
    def flush(self):
        # Wait for all queued documents to be processed
        self.id_queue.join()
        self.doc_queue.join()
    
    def shutdown(self):
        self.shutdown.set()
        self.flush()

Memory Management and Caching​

Content Caching​

// Rust - Content caching layer
use lru::LruCache;
use std::sync::Mutex;
use std::num::NonZeroUsize;

pub struct CachedContentStore<S: ContentStore> {
    store: S,
    cache: Mutex<LruCache<DocId, (String, HashMap<String, String>)>>,
}

impl<S: ContentStore> CachedContentStore<S> {
    pub fn new(store: S, capacity: usize) -> Self {
        Self {
            store,
            cache: Mutex::new(LruCache::new(
                NonZeroUsize::new(capacity).unwrap())
            ),
        }
    }
}

impl<S: ContentStore> ContentStore for CachedContentStore<S> {
    fn get_document(&self, doc_id: DocId) 
        -> Result<(String, HashMap<String, String>)> 
    {
        // Check cache first
        {
            let mut cache = self.cache.lock().unwrap();
            if let Some(doc) = cache.get(&doc_id) {
                return Ok(doc.clone());
            }
        }
        
        // Not in cache, retrieve from store
        let document = self.store.get_document(doc_id)?;
        
        // Add to cache
        {
            let mut cache = self.cache.lock().unwrap();
            cache.put(doc_id, document.clone());
        }
        
        Ok(document)
    }
    
    fn batch_get_documents(&self, doc_ids: &[DocId]) 
        -> Result<Vec<(DocId, String, HashMap<String, String>)>> 
    {
        let mut result = Vec::with_capacity(doc_ids.len());
        let mut missing_ids = Vec::new();
        
        // Get cached documents
        {
            let mut cache = self.cache.lock().unwrap();
            for &doc_id in doc_ids {
                if let Some(doc) = cache.get(&doc_id) {
                    result.push((doc_id, doc.0.clone(), doc.1.clone()));
                } else {
                    missing_ids.push(doc_id);
                }
            }
        }
        
        // Get missing documents from store
        if !missing_ids.is_empty() {
            let missing_docs = self.store.batch_get_documents(&missing_ids)?;
            
            // Add to cache and result
            let mut cache = self.cache.lock().unwrap();
            for (doc_id, content, metadata) in missing_docs {
                cache.put(doc_id, (content.clone(), metadata.clone()));
                result.push((doc_id, content, metadata));
            }
        }
        
        Ok(result)
    }
}
# Python - Content caching layer
import functools
from cachetools import LRUCache
import threading

class CachedContentStore:
    def __init__(self, store, capacity=10000):
        self.store = store
        self.cache = LRUCache(maxsize=capacity)
        self.lock = threading.RLock()
    
    def get_document(self, doc_id):
        # Check cache first
        with self.lock:
            if doc_id in self.cache:
                return self.cache[doc_id]
        
        # Not in cache, retrieve from store
        document = self.store.get_document(doc_id)
        
        # Add to cache
        with self.lock:
            self.cache[doc_id] = document
        
        return document
    
    def batch_get_documents(self, doc_ids):
        result = []
        missing_ids = []
        
        # Get cached documents
        with self.lock:
            for doc_id in doc_ids:
                if doc_id in self.cache:
                    content, metadata = self.cache[doc_id]
                    result.append((doc_id, content, metadata))
                else:
                    missing_ids.append(doc_id)
        
        # Get missing documents from store
        if missing_ids:
            missing_docs = self.store.batch_get_documents(missing_ids)
            
            # Add to cache and result
            with self.lock:
                for doc_id, content, metadata in missing_docs:
                    self.cache[doc_id] = (content, metadata)
                    result.append((doc_id, content, metadata))
        
        return result

use std::fs::File;
use std::io::{BufReader, BufWriter, Read, Seek, SeekFrom, Write};
use std::path::{Path, PathBuf};

pub struct MmapContentStore {
    content_file: PathBuf,
    index_file: PathBuf,
    mmap: Mmap,
    offsets: HashMap<DocId, (usize, usize)>, // (offset, length)
}

impl MmapContentStore {
    pub fn new<P: AsRef<Path>>(content_file: P, index_file: P) -> Result<Self> {
        // Open the content file
        let file = File::open(&content_file)?;
        let mmap = unsafe { MmapOptions::new().map(&file)? };
        
        // Load the offsets from the index file
        let index_file = File::open(&index_file)?;
        let reader = BufReader::new(index_file);
        let offsets: HashMap<DocId, (usize, usize)> = bincode::deserialize_from(reader)?;
        
        Ok(Self {
            content_file: content_file.as_ref().to_path_buf(),
            index_file: index_file.as_ref().to_path_buf(),
            mmap,
            offsets,
        })
    }
}

impl ContentStore for MmapContentStore {
    fn get_document(&self, doc_id: DocId) 
        -> Result<(String, HashMap<String, String>)> 
    {
        // Get the document offset and length
        let (offset, length) = self.offsets.get(&doc_id)
            .ok_or_else(|| Error::DocumentNotFound(doc_id))?;
        
        // Read the document content
        let content_bytes = &self.mmap[*offset..*offset + *length];
        let (content, metadata): (String, HashMap<String, String>) = 
            bincode::deserialize(content_bytes)?;
        
        Ok((content, metadata))
    }
}

import mmap
import os
import json
import struct

class MmapContentStore:
    def __init__(self, content_file, index_file):
        # Open the content file
        self.content_fd = open(content_file, 'rb')
        self.mmap = mmap.mmap(self.content_fd.fileno(), 0, access=mmap.ACCESS_READ)
        
        # Load the offsets from the index file
        with open(index_file, 'rb') as f:
            index_data = f.read()
            
        # Parse the index (format: doc_id(8 bytes), offset(8 bytes), length(8 bytes))
        self.offsets = {}
        for i in range(0, len(index_data), 24):
            doc_id = struct.unpack('<Q', index_data[i:i+8])[0]
            offset = struct.unpack('<Q', index_data[i+8:i+16])[0]
            length = struct.unpack('<Q', index_data[i+16:i+24])[0]
            self.offsets[doc_id] = (offset, length)
    
    def get_document(self, doc_id):
        # Get the document offset and length
        if doc_id not in self.offsets:
            raise KeyError(f"Document {doc_id} not found")
            
        offset, length = self.offsets[doc_id]
        
        # Read the document content
        self.mmap.seek(offset)
        data = self.mmap.read(length)
        
        # Parse the document (format: content length(4 bytes), content, metadata as JSON)
        content_length = struct.unpack('<I', data[:4])[0]
        content = data[4:4+content_length].decode('utf-8')
        metadata_json = data[4+content_length:].decode('utf-8')
        metadata = json.loads(metadata_json)
        
        return content, metadata
    
    def close(self):
        self.mmap.close()
        self.content_fd.close()

use std::time::{Duration, Instant};

pub struct IndexingMetrics {
    start_time: Instant,

    token_count: AtomicUsize,

    indexing_time_ns: AtomicU64,
    batch_count: AtomicUsize,

impl IndexingMetrics {
    pub fn new() -> Self {
        Self {
            start_time: Instant::now(),
            document_count: AtomicUsize::new(0),
            token_count: AtomicUsize::new(0),
            processing_time_ns: AtomicU64::new(0),
            indexing_time_ns: AtomicU64::new(0),
            batch_count: AtomicUsize::new(0),
        }
    }
    
    pub fn record_batch(&self, doc_count: usize, token_count: usize,
                         processing_time: Duration, indexing_time: Duration) {
        self.document_count.fetch_add(doc_count, Ordering::Relaxed);
        self.token_count.fetch_add(token_count, Ordering::Relaxed);
        self.processing_time_ns.fetch_add(processing_time.as_nanos() as u64, 
                                          Ordering::Relaxed);
        self.indexing_time_ns.fetch_add(indexing_time.as_nanos() as u64, 
                                        Ordering::Relaxed);
        self.batch_count.fetch_add(1, Ordering::Relaxed);
    }
    
    pub fn summary(&self) -> String {
        let elapsed = self.start_time.elapsed();
        let doc_count = self.document_count.load(Ordering::Relaxed);
        let token_count = self.token_count.load(Ordering::Relaxed);
        let processing_time_ns = self.processing_time_ns.load(Ordering::Relaxed);
        let indexing_time_ns = self.indexing_time_ns.load(Ordering::Relaxed);
        let batch_count = self.batch_count.load(Ordering::Relaxed);
        
        // Calculate rates
        let docs_per_sec = if elapsed.as_secs() > 0 {
            doc_count as f64 / elapsed.as_secs_f64()
        } else {
            0.0
        };
        
        let tokens_per_sec = if elapsed.as_secs() > 0 {
            token_count as f64 / elapsed.as_secs_f64()
        } else {
            0.0
        };
        
        let avg_batch_size = if batch_count > 0 {
            doc_count as f64 / batch_count as f64
        } else {
            0.0
        };
        
        format!(
            "Indexing Metrics:\n\
             - Documents: {}\n\
             - Tokens: {}\n\
             - Elapsed Time: {:.2?}\n\
             - Docs/sec: {:.2}\n\
             - Tokens/sec: {:.2}\n\
             - Avg Batch Size: {:.2}\n\
             - Processing Time: {:.2?}\n\
             - Indexing Time: {:.2?}",
            doc_count,
            token_count,
            elapsed,
            docs_per_sec,
            tokens_per_sec,
            avg_batch_size,
            Duration::from_nanos(processing_time_ns),
            Duration::from_nanos(indexing_time_ns)
        )
    }
}

            elapsed = time.time() - self.start_time

            docs_per_sec = self.document_count / elapsed if elapsed > 0 else 0
            tokens_per_sec = self.token_count / elapsed if elapsed > 0 else 0

            return {
                "documents": self.document_count,
                "tokens": self.token_count,
                "elapsed_seconds": elapsed,

import time
import threading
import os

class DynamicTuner:
    def __init__(self, metrics, min_threads=1, max_threads=None, 
                 min_batch_size=100, max_batch_size=100000):
        self.metrics = metrics
        self.last_adjustment = time.time()
        self.adjustment_interval = 30  # seconds
        
        # Thread count limits
        self.min_threads = min_threads
        self.max_threads = max_threads or os.cpu_count()
        
        # Batch size limits
        self.min_batch_size = min_batch_size
        self.max_batch_size = max_batch_size
        
        # Current values
        self.current_threads = min(os.cpu_count(), self.max_threads)
        self.current_batch_size = 1000
        
        # Lock for thread safety
        self.lock = threading.RLock()
    
    def check_and_adjust(self):
        with self.lock:
            if time.time() - self.last_adjustment < self.adjustment_interval:
                return self.current_threads, self.current_batch_size
            
            # Get current metrics
            summary = self.metrics.summary()
            current_rate = summary['docs_per_sec']
            
            # Adjust thread count based on CPU usage
            # This is a simple heuristic and could be improved
            new_threads = self.current_threads
            if summary['processing_time'] > summary['indexing_time'] * 1.5:
                # If processing is the bottleneck, increase threads
                new_threads = min(self.current_threads + 2, self.max_threads)
            elif summary['indexing_time'] > summary['processing_time'] * 1.5:
                # If indexing is the bottleneck, possibly reduce threads
                new_threads = max(self.current_threads - 1, self.min_threads)
            
            # Adjust batch size based on throughput
            new_batch_size = self.current_batch_size
            if current_rate < 100:  # Low throughput
                # Try smaller batches for more frequent commits
                new_batch_size = max(self.current_batch_size // 2, self.min_batch_size)
            elif current_rate > 1000:  # High throughput
                # Try larger batches for efficiency
                new_batch_size = min(self.current_batch_size * 2, self.max_batch_size)
            
            # Update current values
            self.current_threads = new_threads
            self.current_batch_size = new_batch_size
            self.last_adjustment = time.time()
            
            print(f"Adjusted parameters: threads={new_threads}, batch_size={new_batch_size}")
            return new_threads, new_batch_size

// Rust - Complete high-throughput indexing pipeline
use std::sync::{Arc, Mutex};
use std::thread;
use std::time::Instant;
use rayon::prelude::*;

pub struct IndexingPipeline {
    content_store: Arc<dyn ContentStore + Send + Sync>,
    index: Arc<Mutex<Index>>,
    batch_size: usize,
    num_threads: usize,
    metrics: Arc<IndexingMetrics>,
}

impl IndexingPipeline {
    pub fn new(
        content_store: Arc<dyn ContentStore + Send + Sync>,
        index: Index,
        batch_size: usize,
        num_threads: usize
    ) -> Self {
        Self {
            content_store,
            index: Arc::new(Mutex::new(index)),
            batch_size,
            num_threads,
            metrics: Arc::new(IndexingMetrics::new()),
        }
    }
    
    pub fn index_collection<I>(&self, doc_ids: I) -> Result<()>
    where
        I: IntoIterator<Item = DocId> + Send,
        I::IntoIter: Send,
    {
        let doc_ids: Vec<_> = doc_ids.into_iter().collect();
        let chunks: Vec<_> = doc_ids.chunks(self.batch_size).collect();
        
        // Enable bulk loading mode
        {
            let mut index = self.index.lock().unwrap();
            if let Some(storage) = index.storage_as_rocksdb_mut() {
                storage.set_bulk_loading_mode(true);
            }
        }
        
        println!("Starting indexing of {} documents in {} chunks", 
                 doc_ids.len(), chunks.len());
        
        // Process each chunk
        chunks.par_iter()
            .with_max_len(1)  // Process one chunk per thread
            .for_each(|chunk| {
                let start = Instant::now();
                
                // Retrieve documents for this chunk
                let processing_start = Instant::now();
                let documents = match self.content_store.batch_get_documents(chunk) {
                    Ok(docs) => docs,
                    Err(e) => {
                        eprintln!("Error retrieving documents: {}", e);
                        return;
                    }
                };
                let processing_time = processing_start.elapsed();
                
                // Count tokens
                let token_count = documents.iter()
                    .map(|(_, content, _)| content.split_whitespace().count())
                    .sum();
                
                // Index the documents
                let indexing_start = Instant::now();
                let mut index_guard = match self.index.lock() {
                    Ok(guard) => guard,
                    Err(e) => {
                        eprintln!("Error acquiring index lock: {}", e);
                        return;
                    }
                };
                
                if let Err(e) = index_guard.add_documents_batch(&documents) {
                    eprintln!("Error indexing documents: {}", e);
                }
                
                let indexing_time = indexing_start.elapsed();
                
                // Record metrics
                self.metrics.record_batch(
                    documents.len(),
                    token_count,
                    processing_time,
                    indexing_time
                );
                
                println!("Processed chunk in {:?}: {} documents, {} tokens",
                         start.elapsed(), documents.len(), token_count);
            });
        
        // Disable bulk loading mode and optimize
        let mut index = self.index.lock().unwrap();
        if let Some(storage) = index.storage_as_rocksdb_mut() {
            storage.set_bulk_loading_mode(false);
        }
        
        // Print final metrics
        println!("{}", self.metrics.summary());
        
        Ok(())
    }
}

# Python - Complete high-throughput indexing pipeline
import concurrent.futures
import threading
import time
import os
import math

class IndexingPipeline:
    def __init__(self, content_store, index, batch_size=1000, num_threads=None):
        self.content_store = content_store
        self.index = index
        self.batch_size = batch_size
        self.num_threads = num_threads or os.cpu_count()
        self.metrics = IndexingMetrics()
    
    def index_collection(self, doc_ids):
        # Convert to list if it's an iterator
        doc_ids = list(doc_ids)
        
        # Split into chunks
        chunk_size = self.batch_size
        chunks = [doc_ids[i:i+chunk_size] for i in range(0, len(doc_ids), chunk_size)]
        
        print(f"Starting indexing of {len(doc_ids)} documents in {len(chunks)} chunks")
        
        # Process chunks in parallel
        with concurrent.futures.ThreadPoolExecutor(max_workers=self.num_threads) as executor:
            futures = [executor.submit(self._process_chunk, chunk) for chunk in chunks]
            
            # Wait for all futures to complete
            for future in concurrent.futures.as_completed(futures):
                try:
                    future.result()
                except Exception as e:
                    print(f"Error processing chunk: {e}")
        
        # Print final metrics
        self.metrics.print_summary()
    
    def _process_chunk(self, chunk):
        start_time = time.time()
        
        # Retrieve documents for this chunk
        processing_start = time.time()
        try:
            documents = self.content_store.batch_get_documents(chunk)
        except Exception as e:
            print(f"Error retrieving documents: {e}")
            return
        
        processing_time = time.time() - processing_start
        
        # Count tokens (simple whitespace tokenization)
        token_count = sum(len(content.split()) for _, content, _ in documents)
        
        # Convert to format expected by index
        batch = [(doc_id, {"content": content, **metadata}) for doc_id, content, metadata in documents]
        
        # Index the documents
        indexing_start = time.time()
        self.index.add_documents_with_fields_parallel(batch)
        indexing_time = time.time() - indexing_start
        
        # Record metrics
        self.metrics.record_batch(
            len(documents),
            token_count,
            processing_time,
            indexing_time
        )
        
        elapsed = time.time() - start_time
        print(f"Processed chunk in {elapsed:.2f}s: {len(documents)} documents, {token_count} tokens")

Conclusion​

By implementing the strategies and patterns outlined in this guide, you can achieve maximum indexing throughput while maintaining separation between your inverted index and content storage. The key principles are:

  1. Optimize RocksDB - Use WriteBatch and bulk loading mode
  2. Process in Batches - Consolidate operations for efficiency
  3. Parallelize - Use multiple threads for document retrieval and processing
  4. Cache Strategically - Use memory efficiently for frequently accessed content
  5. Monitor Performance - Collect metrics and adjust parameters dynamically

Remember that the optimal configuration will depend on your specific hardware, data characteristics, and requirements. Use the performance monitoring tools to identify bottlenecks and tune accordingly.
