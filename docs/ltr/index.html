Learning to Rank (LTR)

The fast-inverted-index library now supports a Learning to Rank (LTR) approach to document scoring and retrieval, providing more flexibility and potentially better results than traditional TF-IDF and BM25 scoring methods.

Overview​

Learning to Rank is a machine learning technique that uses multiple features to score and rank documents for a given query. Unlike traditional scoring methods like TF-IDF and BM25 which use fixed formulas, LTR can:

  1. Combine multiple signals (features) with learned weights
  2. Adapt to specific user preferences and search patterns
  3. Incorporate domain-specific relevance signals
  4. Improve over time with feedback

The current LTR implementation in fast-inverted-index provides:

  1. A linear model that combines multiple features with configurable weights
  2. Feature extraction functionality to support training external ML models
  3. Integration with the existing search API

Using LTR​

Basic Usage​

Using LTR is as simple as specifying "ltr" as the ranking method:

from fast_inverted_index import Index, RankingMethods

# Create an index
index = Index(storage_path="path/to/index")

# Add documents...

# Search using LTR
results = index.search("query text", ranking_method="ltr")

Features Used in LTR Scoring​

The default LTR implementation uses a linear combination of the following features:

  • BM25 score: The basic BM25 relevance score
  • TF-IDF score: The TF-IDF relevance score
  • Exact match: Ratio of query terms that appear in the document
  • Title match: How many query terms match in the title
  • Tag match: How many query terms match in document tags
  • Recency: How recent the document is (newer documents score higher)
  • Length score: Score based on document length relative to average document length

Each feature has a configurable weight in the model.

Customizing Field Boosts​

You can influence the LTR model by providing custom field boosts:

field_boosts = {
    "title": 3.0,     # Higher weight for title matches
    "tags": 2.0,      # Higher weight for tag matches
    "content": 1.0,   # Base weight for content
    "category": 0.8,  # Lower weight for category
}

results = index.search("query", ranking_method="ltr", boost_fields=field_boosts)

Advanced: Feature Extraction for Custom Models​

For advanced users who want to train their own LTR models, the library provides feature extraction capabilities:

// This is available in the Rust API
let features = index.extract_ltr_features(&query, None)?;

// Features contain term, document, and query-document interaction features
for feature in features {
    println!("Doc ID: {}", feature.doc_id);
    println!("Term features: {:?}", feature.term_features);
    println!("Document features: {:?}", feature.doc_features);
    println!("Interaction features: {:?}", feature.interaction_features);
}

The extracted features can be used to train custom ranking models with libraries like XGBoost, LightGBM, or scikit-learn.

How LTR Works Internally​

  1. When a query is executed with the LTR ranking method:

    • The index first retrieves matching documents using standard boolean retrieval
    • Both BM25 and TF-IDF basic scores are calculated for all matching documents
    • Additional features are extracted (title matches, tag matches, recency, etc.)
    • All features are weighted and combined into a final score
    • Documents are sorted by their final score

  2. The default LTR model is a linear combination of features with pre-configured weights. The weights are set to provide reasonable out-of-the-box performance, but can be customized for specific use cases.

Performance Considerations​

  • LTR may be more computationally intensive than basic BM25 or TF-IDF as it calculates multiple features per document
  • For very large result sets, consider using BM25 for initial retrieval and LTR for re-ranking the top results
  • Feature extraction is the most expensive part of the LTR process, especially for large documents

Future Directions​

Planned improvements for the LTR implementation include:

  1. Support for external LTR models (e.g., ONNX format models)
  2. User feedback integration for relevance improvements
  3. More sophisticated features, including semantic/embedding-based features
  4. Python API for feature extraction and model training
  5. Click-through rate (CTR) and conversion optimization
