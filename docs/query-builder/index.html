Query Builder API

The QueryBuilder API provides a convenient way to construct and execute queries against the Fast Inverted Index from Python.

Overview​

The QueryBuilder class is a helper class that simplifies the process of creating queries with multiple terms and applying boolean operations like AND and OR. It is especially useful for programmatically building queries based on user input or other dynamic sources.

Basic Usage​

from fast_inverted_index import Index, QueryBuilder

# Create an index and add some documents
index = Index()
index.add_document(1, "Rust programming language", {"tags": ["rust", "programming"]})
index.add_document(2, "Python programming language", {"tags": ["python", "programming"]})

# Create a QueryBuilder and add terms
builder = QueryBuilder(index)
builder.add_term("rust")

# Execute an OR query
doc_ids = builder.build_or_query()
# Returns [1]

# Add another term
builder.add_term("python")

# Execute an OR query with both terms
doc_ids = builder.build_or_query()
# Returns [1, 2]

# Execute an AND query with both terms
doc_ids = builder.build_and_query()
# Returns [] (no document contains both terms)

# Clear all terms
builder.clear()

API Reference​

Constructor​

QueryBuilder(index: Index)

Creates a new QueryBuilder instance associated with the given index.

Parameters:

  • index: The Index instance to query against

Methods​

add_term(term: str) -> QueryBuilder​

Adds a single term to the query.

Parameters:

  • term: The term to add

Returns:

  • The builder instance for method chaining

build_and_query() -> List[int]​

Builds and executes an AND query with all added terms.

Returns:

  • A list of document IDs that match all terms in the query

build_or_query() -> List[int]​

Builds and executes an OR query with all added terms.

Returns:

  • A list of document IDs that match at least one term in the query

clear() -> QueryBuilder​

Clears all terms from the builder.

Returns:

  • The builder instance for method chaining

Implementation Details​

The QueryBuilder is a thin wrapper around the Index's and_query and or_query methods. When you call build_and_query() or build_or_query(), the builder passes all collected terms to the corresponding method on the Index.

Under the hood, the Index creates a Rust Query object with the appropriate boolean operator and executes it against the index. The results are returned as a list of document IDs.

Alternative Approaches​

If you need more advanced query capabilities, consider using:

  1. The search method on the Index, which returns document IDs with scores:

    # Direct search with AND query
    results = index.search("term1 term2")  # AND query
    # Returns [(doc_id, score), ...]

    # Direct search with OR query
    results = index.search("term1 OR term2")  # OR query
    # Returns [(doc_id, score), ...]

  2. The QueryNode API for complex queries with field-specific searches, boosting, and negation:

    from fast_inverted_index import QueryNode, QueryExecutionParams

    # Build a complex query
    query = QueryNode.OR([
        QueryNode.term("title", "rust", boost=2.0),
        QueryNode.term("content", "programming")
    ])

    # Execute with custom parameters
    params = QueryExecutionParams(scoring_algorithm="bm25l", limit=10)
    result = index.execute_query(query, params)

Query Parsing Rules​

When using string-based queries (via the search method), it's important to understand the behavior of the query parser:

  1. Default Operator: Spaces between terms create an AND relationship (all terms must match)
  2. OR Operator: The OR keyword creates an OR relationship between terms
  3. Parser Behavior: The query parser has specific rules for handling OR operators

Examples:

  • "term1 term2" → Documents containing both term1 AND term2
  • "term1 OR term2" → Documents containing either term1 OR term2
  • "term1 OR term2 term3" → Only matches documents containing term3 AND either term1 OR term2
  • "term1 term2 OR term3" → Documents containing either (term1 AND term2) OR term3

Important Notes:

  1. The parser doesn't support parentheses for explicit grouping
  2. The parser treats OR operations with higher precedence than AND
  3. For complex queries with explicit grouping, field-specific searches, or advanced operators, use the QueryNode API
  4. When writing queries, keep them simple and test them to verify behavior

If you need more precise control, use the QueryBuilder to construct boolean queries programmatically:

# Build an AND query
builder = QueryBuilder(index)
builder.add_term("term1")
builder.add_term("term2")
doc_ids = builder.build_and_query()

Examples​

Building a Search Form​

def search_documents(index, query_string, fields=None, use_or=False):
    # Split the query string into terms
    terms = query_string.split()
    
    # Create a builder
    builder = QueryBuilder(index)
    
    # Add each term
    for term in terms:
        builder.add_term(term)
    
    # Execute query based on the operator choice
    if use_or:
        return builder.build_or_query()
    else:
        return builder.build_and_query()

Combining Multiple QueryBuilders​

# Create builders for different field queries
title_builder = QueryBuilder(index)
title_builder.add_term("rust")

content_builder = QueryBuilder(index)
content_builder.add_term("systems")
content_builder.add_term("programming")

# Get results from each builder
title_docs = title_builder.build_and_query()
content_docs = content_builder.build_and_query()

# Combine the results (union of sets)
combined_docs = list(set(title_docs) | set(content_docs))

Thread Safety​

The QueryBuilder class is not thread-safe. If you need to use the same index from multiple threads, each thread should create its own QueryBuilder instance.
