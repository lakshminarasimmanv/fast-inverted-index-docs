Schema and Multi-Field Indexing

The Schema system allows you to define structured document formats with multiple field types and custom analyzers. This enables more sophisticated document indexing and field-specific searching with relevance control.

Overview​

The Schema system provides:

  • A flexible document model with typed fields
  • Support for multiple field types (text, keyword, numeric, etc.)
  • Per-field analyzers for customized text processing
  • Field-specific inverted indexes
  • Configurable field boosting for relevance control
  • Schema serialization for persistence

Field Types​

The following field types are supported:

TypeDescriptionAnalyzerUse Case
TextTokenized text with analysisStandardGeneral content, descriptions
KeywordNon-tokenized exact matchKeywordIDs, categories, exact matching
NumericInteger or float valuesNoneCounts, ratings, metrics
DateISO format dates/timestampsNoneTimestamps, calendars
BooleanTrue/false valuesNoneFlags, status indicators

Rust API​

Creating a Schema​

use fast_inverted_index::schema::{Schema, SchemaBuilder, FieldSchema, FieldType, AnalyzerType};

// Create a schema using the builder
let schema = SchemaBuilder::new()
    .add_field(FieldSchema::text("title").with_boost(5.0))
    .add_field(FieldSchema::text("content").with_boost(1.0))
    .add_field(FieldSchema::keyword("tags").with_boost(3.0))
    .add_field(FieldSchema::numeric("rating"))
    .add_field(FieldSchema::date("created_at"))
    .add_field(FieldSchema::boolean("is_active"))
    .default_field("content")
    .build();

// Create a schema manually
let mut schema = Schema::new();
schema.add_field(FieldSchema::new("title", FieldType::Text).with_boost(5.0));
schema.add_field(FieldSchema::new("content", FieldType::Text).with_boost(1.0));
schema.set_default_field("content");

// Save schema to a file
schema.save_to_file("schema.json").unwrap();

// Load schema from a file
let loaded_schema = Schema::load_from_file("schema.json").unwrap();

Creating an Index with Schema​

use fast_inverted_index::{Index, IndexBuilder};
use std::collections::HashMap;

// Create an index with the schema
let mut index = IndexBuilder::new()
    .with_storage_path("/path/to/index")
    .with_schema(schema)
    .build()
    .unwrap();

// Add a document with fields
let mut fields = HashMap::new();
fields.insert("title".to_string(), "Document Title".to_string());
fields.insert("content".to_string(), "This is the document content".to_string());
fields.insert("tags".to_string(), "example documentation".to_string());
fields.insert("rating".to_string(), "4.5".to_string());
fields.insert("created_at".to_string(), "2025-05-01T12:00:00Z".to_string());
fields.insert("is_active".to_string(), "true".to_string());

index.add_document_with_fields(1, fields).unwrap();

Searching by Field​

use fast_inverted_index::query::{QueryNode, QueryExecutionParams, ScoringAlgorithm};
use std::collections::HashMap;

// Search in a specific field
let title_query = QueryNode::term("title", "document");
let result = index.execute_query(&title_query, None).unwrap();

// Search multiple fields with different boosts
let multi_field_query = QueryNode::or(vec![
    QueryNode::term_with_boost("title", "document", 2.0),
    QueryNode::term_with_boost("content", "document", 1.0),
    QueryNode::term_with_boost("tags", "document", 1.5)
]);

// Create execution parameters with field boosts
let mut params = QueryExecutionParams::default();
params.scoring_algorithm = ScoringAlgorithm::BM25;

// Add global field boosts (applied on top of per-query boosts)
let mut field_boosts = HashMap::new();
field_boosts.insert("title".to_string(), 1.5);
field_boosts.insert("tags".to_string(), 1.2);
params.field_boosts = field_boosts;

let result = index.execute_query(&multi_field_query, Some(params)).unwrap();

Python API​

Creating a Schema​

from fast_inverted_index import Schema, FieldSchema, FieldType, AnalyzerType

# Create a schema
schema = Schema()

# Add fields with different types
schema.add_field(FieldSchema.text("title").with_boost(5.0))
schema.add_field(FieldSchema.text("content").with_boost(1.0))
schema.add_field(FieldSchema.keyword("tags").with_boost(3.0))
schema.add_field(FieldSchema.numeric("rating"))
schema.add_field(FieldSchema.date("created_at"))
schema.add_field(FieldSchema.boolean("is_active"))

# Set default field for queries that don't specify a field
schema.set_default_field("content")

# Get schema information
print(f"Schema has {schema.len()} fields")
print(f"Field names: {schema.field_names()}")

# Save schema to a file
schema.save_to_file("schema.json")

# Load schema from a file
loaded_schema = Schema.load_from_file("schema.json")

Creating an Index with Schema​

from fast_inverted_index import Index

# Create an index with the schema
index = Index(storage_path="/path/to/index", schema=schema)

# Add a document with fields
doc_fields = {
    "title": "Document Title",
    "content": "This is the document content",
    "tags": "example documentation",
    "rating": "4.5",
    "created_at": "2025-05-01T12:00:00Z",
    "is_active": "true"
}

index.add_document_with_fields(1, doc_fields)

Searching by Field​

from fast_inverted_index import QueryNode, QueryExecutionParams

# Search in a specific field with the convenience method
title_results = index.search_field("title", "document", ranking_method="bm25", limit=10)

# Get the first matching document
if title_results:
    doc_id, score = title_results[0]
    doc = index.get_document(doc_id)
    print(f"Found document: {doc.get('title', 'Untitled')} (Score: {score:.4f})")

# Search multiple fields with QueryNode API
multi_field_query = QueryNode.OR([
    QueryNode.term("title", "document", boost=2.0),
    QueryNode.term("content", "document", boost=1.0),
    QueryNode.term("tags", "document", boost=1.5)
])

# Create execution parameters with field boosts
params = QueryExecutionParams(
    scoring_algorithm="bm25",
    limit=10,
    field_boosts={
        "title": 1.5,
        "tags": 1.2
    }
)

# Execute the query
results = index.execute_query(multi_field_query, params)

# Process the results
for doc_id, score in results.scored_docs:
    doc = index.get_document(doc_id)
    print(f"Document ID: {doc_id}, Score: {score:.4f}")
    print(f"Title: {doc.get('title', 'Untitled')}")
    print(f"Content: {doc.get('content', 'No content')[:50]}...")

Field Boosting with Simple Search API​

# Define field boosts
boost_fields = {
    "title": 5.0,    # Title matches are 5x more important
    "content": 1.0,  # Content matches have normal importance
    "tags": 3.0      # Tag matches are 3x more important
}

# Search with boosts across all fields
results = index.search("document", ranking_method="bm25", boost_fields=boost_fields, limit=10)

Advanced Examples​

Complete Schema Definition Example​

from fast_inverted_index import Schema, FieldSchema, FieldType, AnalyzerType

# Create a schema for a blog post
schema = Schema()

# Add fields with explanations
schema.add_field(FieldSchema.text("title")
    .with_boost(5.0)
    .with_analyzer(AnalyzerType.STANDARD))
# Title field: Important for search relevance, gets 5x boost

schema.add_field(FieldSchema.text("content")
    .with_boost(1.0)
    .with_analyzer(AnalyzerType.STANDARD))
# Content field: Main text, normal importance

schema.add_field(FieldSchema.keyword("tags")
    .with_boost(3.0)
    .with_analyzer(AnalyzerType.KEYWORD))
# Tags field: Important for categorization, uses keyword analyzer for exact matching

schema.add_field(FieldSchema.text("author")
    .with_boost(2.0))
# Author field: Author name, gets 2x boost

schema.add_field(FieldSchema.keyword("category")
    .with_boost(1.5))
# Category field: Blog category, keyword field for exact matching

schema.add_field(FieldSchema.date("published_at"))
# Publication date: For sorting and filtering

schema.add_field(FieldSchema.numeric("read_time"))
# Read time in minutes: For filtering articles by length

schema.add_field(FieldSchema.boolean("is_published"))
# Publication status: For filtering published/draft posts

# Set default field for queries
schema.set_default_field("content")

Complex Queries Using Multiple Fields​

from fast_inverted_index import QueryNode, QueryExecutionParams, QueryBound, Value

# Find Python tutorials by Jane Smith
complex_query = QueryNode.AND([
    # Document must be about Python
    QueryNode.OR([
        QueryNode.term("title", "python", boost=5.0),
        QueryNode.term("content", "python", boost=1.0),
        QueryNode.term("tags", "python", boost=3.0)
    ]),
    # Must be authored by Jane Smith
    QueryNode.term("author", "jane smith"),
    # Must be a tutorial
    QueryNode.term("tags", "tutorial"),
    # Must be published
    QueryNode.term("is_published", "true")
])

# Execute the query
params = QueryExecutionParams(
    scoring_algorithm="bm25l",
    limit=10,
    field_boosts={
        "title": 2.0,
        "tags": 1.5
    }
)
results = index.execute_query(complex_query, params)

# Process matching documents
for doc_id, score in results.scored_docs:
    doc = index.get_document(doc_id)
    print(f"Document {doc_id}: {doc.get('title', 'Untitled')} (Score: {score:.4f})")
    print(f"  Author: {doc.get('author', 'Unknown')}")
    print(f"  Tags: {doc.get('tags', 'None')}")
    print(f"  Read time: {doc.get('read_time', 'Unknown')} minutes")
    print(f"  Published: {doc.get('published_at', 'Unknown')}")
    print()

# Find recent data science articles with short read time
date_threshold = "2025-01-01T00:00:00Z"
read_time_query = QueryNode.AND([
    # Must be in data science category
    QueryNode.term("category", "data-science"),
    # Published after January 1, 2025
    QueryNode.range(
        "published_at",
        QueryBound.included(Value.string(date_threshold)),
        QueryBound.unbounded()
    ),
    # Read time under 15 minutes
    QueryNode.range(
        "read_time",
        QueryBound.unbounded(),
        QueryBound.included(Value.integer(15))
    )
])

time_results = index.execute_query(read_time_query)

Best Practices​

  1. Define Appropriate Field Types: Use the most specific field type for each field (text for full-text search, keyword for exact matching, numeric for numbers, etc.)

  2. Set Appropriate Boosts: Give higher boosts to more important fields (title, tags) and lower boosts to less important fields (content)

  3. Choose the Right Analyzer: Use the standard analyzer for normal text, the keyword analyzer for exact matches, and the simple analyzer for minimal processing

  4. Set a Default Field: Always set a default field for queries that don't specify a field, typically the main content field

  5. Store Fields Properly: Use add_document_with_fields() for multi-field documents instead of the legacy add_document() with metadata

  6. Ensure Field Consistency: Keep field types consistent across all documents; don't mix types for the same field

  7. Use Field-Specific Queries: When possible, direct queries to specific fields rather than searching all fields

  8. Combine Field Boosts with Query Boosts: Use both the field's default boost and query-specific boosts for fine-grained relevance control

Common Issues and Solutions​

Field Values Not Appearing in Document Metadata​

If field values aren't appearing when retrieving documents:

# Ensure you're using add_document_with_fields and not just add_document
index.add_document_with_fields(doc_id, fields)

# Verify fields when retrieving the document
doc = index.get_document(doc_id)
for field in schema.field_names():
    if field in doc:
        print(f"{field}: {doc[field]}")
    else:
        print(f"{field}: Not found")

Query Returns Zero or Low Scores​

If your field queries are returning zero or unexpectedly low scores:

# Ensure you're searching the correct field names
results = index.search_field("title", "term", ranking_method="bm25", limit=10)

# Check if you need to use the exact term for keyword fields (case-sensitive)
keyword_results = index.search_field("category", "Exact-Category-Name", ranking_method="bm25")

# Try using field boosting to increase scores
boosted_results = index.search("term", ranking_method="bm25", 
                              boost_fields={"title": 5.0, "tags": 3.0})

Text Field Search Considerations​

When searching text fields, be aware of how text analysis affects your queries:

  1. Tokenization: Text fields are tokenized into individual terms. For example, "Jane Smith" becomes ["jane", "smith"].

  2. Case Sensitivity: Terms are typically lowercased during indexing. When searching, your query is also lowercased.

  3. Single vs. Multi-term Searches:

    • When using search_field(), it creates a term query looking for an exact token match:

      # This searches for the token "jane", not the full name "Jane Smith"
      results = index.search_field("author", "Jane")

    • For multi-term queries, use the QueryNode API with phrase queries:

      # This searches for the phrase "jane smith" (with terms in that order)
      query = QueryNode.phrase("author", ["jane", "smith"], slop=0)
      results = index.execute_query(query, params)

  4. Partial Matching: For partial term matches, use prefix queries:

    # This matches terms starting with "ja"
    query = QueryNode.prefix("author", "ja")
    results = index.execute_query(query, params)

  5. Using Keyword Fields: For exact string matching without tokenization, use keyword fields instead of text fields:

    # Define a keyword field
    schema.add_field(FieldSchema.keyword("exact_name"))

    # Add document with an exact name field
    doc_fields = {"exact_name": "Jane Smith"}
    index.add_document_with_fields(1, doc_fields)

    # Search for the exact string match
    results = index.search_field("exact_name", "Jane Smith")

Schema Not Serializing or Deserializing Correctly​

If you have issues with schema persistence:

# Ensure the schema directory exists before saving
import os
os.makedirs(os.path.dirname("path/to/schema.json"), exist_ok=True)

# Use absolute paths for schema files
schema.save_to_file(os.path.abspath("path/to/schema.json"))

# Check for file permissions
try:
    schema.save_to_file("path/to/schema.json")
except Exception as e:
    print(f"Error saving schema: {e}")

# Verify the schema after loading
loaded_schema = Schema.load_from_file("path/to/schema.json")
print(f"Loaded {loaded_schema.len()} fields: {loaded_schema.field_names()}")
